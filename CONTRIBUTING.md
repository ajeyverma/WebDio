# Contributing to WebDio

Thank you for your interest in contributing to WebDio! We welcome contributions of all sizes — bug fixes, documentation improvements, new features, and design or UX suggestions.

## Getting Started

1. Fork the repository and create a feature branch:

```bash
git clone <repository-url>
cd WebDio
git checkout -b feat/your-feature
```

2. Install dependencies:

```bash
npm install
```

3. Run the app in development:

```bash
npm run dev
```

## How to Contribute

- Report bugs by opening an issue with a clear title and reproduction steps.
- For code changes, open a pull request against the `master` branch with a descriptive title and summary of changes.
- Include tests and documentation where applicable.
- Keep commits small and focused. Use semantic commit messages.

## Branching and PR Guidelines

- Base your work on the latest `master` branch.
- Create feature branches prefixed with `feat/`, `fix/`, or `chore/`.
- Include screenshots or short recordings for UI changes.
- Ensure all TypeScript checks pass and the app builds successfully.

## Code Style

- Follow existing code style and file organization.
- Use modern TypeScript and React best practices.
- Run the formatter (Prettier) before committing.

## Tests

- Add unit tests for new logic where feasible.
- If adding UI changes, test critical paths manually and include instructions in the PR.

## Thank you

Thanks for helping make WebDio better! If you'd like guidance on good first issues to work on, open an issue and tag it `help wanted`.
